# 脱敏规则

## 禁止公开内容

| 类型 | 禁止内容 | 替代表达 |
| --- | --- | --- |
| 公司信息 | 真实公司名、部门、老板姓名 | 某工业设备企业、项目负责人 |
| 客户信息 | 客户姓名、手机号、微信、邮箱 | 华东某制造企业采购负责人 |
| 商务信息 | 合同、报价、付款、订单 | 模拟报价区间、虚构成交路径 |
| 账号数据 | 后台截图、真实播放量、投流数据 | 模拟数据、区间数据、趋势描述 |
| 销售线索 | 真实询盘、真实聊天记录 | 虚构线索、模拟对话 |
| 内部资料 | 会议原文、内部文件、复盘纪要 | 公开版方法总结 |
| 产品细节 | 未公开参数、内部成本、供应商信息 | 通用参数维度、公开资料描述 |

## 脱敏方法

1. 名称替换：真实公司改为虚构品牌。
2. 数据替换：真实数据改为模拟区间。
3. 场景抽象：真实客户场景改为行业通用场景。
4. 截图删除：公开版不放真实截图。
5. 对话重写：真实聊天记录改为示例话术。
6. 链路泛化：真实成交过程改为通用流程。

## 虚构数据标注

所有案例文件必须写明：

> 本案例为虚构模拟案例，所有品牌、客户、数据、询盘和结果均为公开展示用示例。

## 发布前自查清单

- 没有真实人名。
- 没有联系方式。
- 没有真实公司名。
- 没有真实截图。
- 没有合同报价。
- 没有账号后台数据。
- 没有可反推真实业务的细节。
