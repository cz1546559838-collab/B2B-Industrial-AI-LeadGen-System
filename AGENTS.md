# AGENTS.md

## 项目角色

本项目是公开版求职作品集，用于展示 B2B 工业品 AI 获客方法和新媒体运营能力。

## 执行规则

1. 所有内容必须脱敏、模拟化、模板化。
2. 不得写入任何真实公司、真实客户、真实账号、真实数据、真实截图或内部文件。
3. 所有 demo case 必须使用“示例品牌A/B/C”“模拟客户A/B/C”“虚构数据”。
4. 输出内容优先保留方法论、流程、模板、复盘框架。
5. 修改文件前先确认是否属于公开版内容。
6. 涉及私有资料时，只能放入 `99_private_not_upload/`，且不得提交到 GitHub。
7. 未经用户明确确认，禁止自动执行 `git init`。
8. 未经用户明确确认，禁止执行 `git commit`、`git push`。
9. 未经用户明确确认，禁止上传 GitHub 或任何公开平台。
10. 每次准备发布前，必须执行公开版敏感信息检查。

## 禁止内容

- 公司真实名称、内部项目名、真实产品报价。
- 客户姓名、手机号、微信、邮箱、公司名。
- 合同、报价单、订单、付款记录、聊天记录。
- 账号后台截图、广告投放数据、真实线索数据。
- 老板会议原文、内部复盘、未公开策略。
- 真实素材引用、真实客户现场截图、真实账号后台截图。

## 发现风险时的处理

如发现以下任意内容，必须停止并提示用户确认：

- 真实公司名
- 客户名
- 手机号
- 微信号
- 邮箱
- 报价
- 合同
- 后台数据
- 真实截图
- 真实素材引用

## 发布前安全检查

1. 检查是否存在真实公司、客户、联系方式和商务信息。
2. 检查是否存在账号后台数据、投流数据、真实截图或真实素材。
3. 检查所有案例是否明确标注虚构模拟。
4. 检查 demo case 是否使用“示例品牌A/B/C”“模拟客户A/B/C”“虚构数据”。
5. 检查 `.gitignore` 是否覆盖私有目录和高风险格式。
6. 注意：`.gitignore` 不能替代安全检查，已被 git 跟踪的文件仍有泄露风险。

## 推荐写法

- 用“某工业设备企业”替代真实公司。
- 用“华东某制造企业采购负责人”替代真实客户。
- 用“模拟数据”替代真实后台数据。
- 用“虚构案例”替代真实项目复盘。
- 用“示例品牌A/B/C”替代任何可能撞名的品牌。
- 用“模拟客户A/B/C”替代真实客户类型。

## 文件维护原则

- 小步修改，不做无关重构。
- 新增内容先检查公开风险。
- 模板尽量可复用。
- 案例必须在开头标注“本案例为虚构模拟案例”。
