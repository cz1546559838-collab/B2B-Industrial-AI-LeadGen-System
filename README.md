# B2B工业品AI获客系统 GitHub作品集

这是一个面向求职展示的公开版作品集项目，用于呈现我在 B2B 工业品内容获客、工业设备客户决策分析、AI 工作流搭建、线索筛选和复盘优化方面的系统化能力。

本项目只展示方法论、流程、模板和虚构模拟案例，不包含任何真实公司资料、客户信息、后台数据、合同报价、真实线索或内部文件。

## 30 秒看懂我能做什么

- 我能把工业设备的技术参数转化成客户看得懂的内容选题。
- 我能围绕短视频、小红书和图文平台搭建 B2B 工业品内容矩阵。
- 我能用 AI 辅助完成选题、脚本、评论分类、线索摘要和日报周报。
- 我能把内容互动转化为可分层、可跟进、可复盘的销售线索。
- 我能把一套获客方法迁移到不同工业品类，例如焊接设备、切割设备、自动化设备和制造业服务。

## 我能为 B2B 工业品企业解决什么问题

| 企业常见问题 | 我的解决思路 | 产出物 |
| --- | --- | --- |
| 产品专业，但客户看不懂 | 从客户场景、痛点和采购角色重写内容表达 | 客户画像、痛点清单、短视频脚本 |
| 账号只会发产品展示，缺少获客结构 | 按引流款、技术流、案例型、问答型搭建内容矩阵 | 内容矩阵表、选题池、发布节奏 |
| 评论和私信杂乱，销售难跟进 | 建立 A/B/C/D 线索分层和跟进摘要 | 评论筛选表、私信追问模板、线索摘要 |
| 运营动作重复，效率低 | 用 AI 辅助选题、脚本、日报周报和复盘 | AI 工作流、日报模板、周报模板 |
| 内容发了但不知道怎么优化 | 用互动质量、线索质量和采购阶段做复盘 | 复盘模板、下一轮优化动作 |

## 项目定位

- 面向行业：B2B 工业品、工业设备、制造业服务商。
- 核心目标：用内容获客方法和 AI 工作流提升选题、脚本、发布、线索筛选、日报周报和复盘效率。
- 展示方式：公开版文档 + 模板 + 模拟案例。
- 版本状态：v1.1 求职作品集版。

## 公开版边界

本仓库适合作为公开作品集展示，但不适合作为企业内部资料仓库。

公开版允许包含：

- 通用方法论
- 流程图和执行框架
- 可复用模板
- 虚构案例
- 示例品牌、模拟客户、虚构数据

公开版禁止包含：

- 公司真实资料
- 客户姓名、手机号、微信、邮箱
- 合同、报价、订单、付款记录
- 账号后台截图和投流数据
- 老板会议原文、内部文件、真实线索
- 任何可以反推真实业务的信息

## 目录说明

- `00_project_overview/`：项目说明、公开版边界和脱敏规则。
- `01_market_research/`：客户画像、竞品账号分析和行业需求拆解模板。
- `02_content_strategy/`：工业品内容矩阵、引流内容、技术流内容和标题模板。
- `03_ai_workflows/`：AI 辅助选题、脚本、线索筛选、日报周报流程。
- `04_templates/`：短视频脚本、评论筛选、私信跟进、日报和复盘模板。
- `05_demo_cases/`：全部为示例品牌、模拟客户和虚构数据的模拟案例。
- `99_private_not_upload/`：私有资料隔离区，默认禁止上传。

## 使用说明

1. 阅读 `00_project_overview/公开版边界.md` 和 `00_project_overview/脱敏规则.md`。
2. 按行业选择 `01_market_research/` 中的模板做公开版分析。
3. 使用 `02_content_strategy/` 搭建内容矩阵。
4. 使用 `03_ai_workflows/` 把重复运营动作流程化。
5. 使用 `04_templates/` 输出可复用工作文档。
6. 参考 `05_demo_cases/` 的虚构案例表达项目经验。
