# B2B工业品AI获客系统 GitHub作品集

一句话定位：这是一个面向求职展示的 B2B 工业品 AI 获客系统作品集，展示如何用客户决策分析、内容矩阵、AI 工作流、线索筛选和复盘优化，把工业品新媒体内容转化为可跟进的销售线索。

本项目是公开版作品集，只包含方法论、流程、模板和虚构模拟案例，不包含真实公司资料、客户信息、联系方式、合同报价、账号后台数据、投流数据、真实线索、内部文件或真实截图。

## Methodology / 方法论能力

### 1. B2B 工业品客户决策分析

工业品客户通常不是单一用户决策，而是由使用者、技术负责人、采购负责人和管理层共同影响。作品集中的客户画像模板和模拟案例，会按角色拆解客户关心的问题，例如效率、稳定性、交付、售后、维护成本和选型风险。

### 2. 内容矩阵设计

内容不是简单展示产品，而是按采购阶段设计。引流款负责扩大曝光，技术流建立专业信任，问答型承接评论问题，案例型展示应用场景，复盘型沉淀运营判断。

### 3. AI 辅助脚本生产

AI 在这里不是替代判断，而是提升重复工作效率。流程包括输入客户画像和痛点，生成选题池和脚本草稿，再由人工复核技术准确性、公开风险和表达方式。

### 4. 线索筛选

评论、私信和表单信息需要先结构化，再交给销售跟进。项目中使用 A/B/C/D 分层：A 类适合优先跟进，B 类需要补充问题，C 类进入内容培育，D 类过滤。

### 5. 复盘优化

复盘不只看曝光，而是看互动质量和线索质量。重点指标包括客户是否主动提供行业、材料、规格、工况、采购阶段，以及内容是否能引导下一步沟通。

### 6. 可迁移到不同工业品类

本作品集的方法不绑定单一产品。只要能拆出客户角色、采购链路、核心顾虑和选型参数，就可以迁移到焊接设备、切割设备、自动化设备、制造业服务等不同工业品类。

## Public Version Safety Notice / 公开版安全声明

本仓库适合作为公开作品集展示，不适合作为企业内部资料仓库。

公开版只允许包含：

- 通用方法论
- 执行流程
- 可复用模板
- 虚构模拟案例
- 示例品牌、模拟客户、虚构数据

公开版不包含：

- 真实公司资料
- 客户信息和联系方式
- 合同、报价、订单、付款记录
- 账号后台数据和投流数据
- 真实线索、真实聊天记录、真实截图
- 老板会议原文、内部文件或未公开策略

所有 demo case 均为虚构模拟案例，不代表任何真实公司、客户或经营数据。
